# Bug Report Template — IEEE 1044 (Software Anomalies Classification)

Use this template to generate the specification document for `fix/` work items. This replaces the SRS for bug fixes, focusing on anomaly documentation rather than requirements elicitation.

## Filling Rules

1. **Steps to reproduce MUST be specific and numbered** — a developer must be able to reproduce the bug from scratch
2. **Expected vs actual behavior MUST reference the original specification** when available
3. **Each acceptance criterion MUST be verifiable** — Given/When/Then format preferred
4. **Severity MUST be classified** using the same scale as the refactoring-severity-guide (Critical/High/Medium/Low)
5. **Evidence MUST include at least one type** — logs, screenshots, or network traces
6. **Impact analysis MUST cover data impact** — is any data being corrupted or lost?
7. **Section 9 is generated by the agent during Phase 1B** — the agent reads code, forms hypotheses, and asks discriminating questions to eliminate them until the root cause is confirmed
