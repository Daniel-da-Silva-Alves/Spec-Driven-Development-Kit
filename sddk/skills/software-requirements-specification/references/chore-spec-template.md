Use this template to generate the specification document for `chore/` work items. This replaces the SRS for maintenance tasks, focusing on infrastructure changes, dependency updates, backward compatibility, and rollback planning.

## Filling Rules

1. **Maintenance type MUST be classified** — Adaptive (environment changed) or Preventive (proactive fix)
2. **Breaking changes MUST be explicitly listed** with migration strategies
3. **Rollback plan is MANDATORY** — every chore must have a revert path
4. **Current state MUST include end-of-life or risk information** — why is the change needed now?
5. **Acceptance criteria MUST include "build passes" and "tests green"** as minimum
6. **Affected features MUST be identified** — no chore is truly isolated
7. **Section 9 is generated by the agent during Phase 1B** — the agent reads changelogs/release notes and greps the codebase for deprecated or removed APIs
